#DiOne配置平台接口说明	

##环境说明
* 环境配置详见【README.md】
##输入参数
* 接口都采用post form-data的方式传递参数
##输出参数
* 1、输出格式统一为json
* 2、正确返回json的形式一般是这样的：
```json
{
    code: 0
}
```
* 3、错误返回json的形式一般是这样的：
```json
{
    "code": "0004",
    "msg": "ApplicationID Illegal",
    "extmsg": "非法的业务Id"
}
```

###查根据set属性查询主机

SetEnviType Set环境类型，选填，传多个,分割	选填
SetServiceStatus Set开放状态，选填，传多个,分割	选填
ModuleName	模块名，选填，传多个,分割	选填
* 响应信息：
成功：

```json
{
    "code": 0,
    "data": [
        {
            "InnerIP": "10.126.84.11",
            "OuterIP": "10.126.84.11",
            "Source": "3",
            "HostID": "2"
        }
    ]
}
```

五、错误信息列表

code	msg	extMsg
* 0001
ApplicationID Illegal
非法的业务id

* 0002
no app
没有业务

* 0004
user name illegal
用户id非法

* 0005
only default app exsit
只有默认业务

* 0006
only right to app
没权利访问业务

* 0011
no host
没有主机

* 0016
invalid input ApplicationID
业务id输入无效
